# gctx Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Package the provided Babashka-based Google Cloud configuration switcher as `gctx` in this NUR repository.

**Architecture:** Keep the command implementation in a standalone executable `gctx.clj`. Package it with `stdenvNoCC.mkDerivation` and `makeWrapper`, adding Babashka, fzf, and Google Cloud CLI to its runtime `PATH`; expose it through the repository package set.

**Tech Stack:** Nix, Babashka/Clojure, shell-based behavior tests

---

## File Structure

- `pkgs/gctx/gctx.clj`: command behavior and CLI parsing.
- `pkgs/gctx/default.nix`: installation, runtime dependency wrapping, and package metadata.
- `pkgs/gctx/test.sh`: isolated behavior tests using fake `gcloud` and `fzf` executables.
- `default.nix`: exports `gctx` from the repository package set.
- `README.md`: lists `gctx` as an available package.

### Task 1: Add failing command behavior tests

**Files:**
- Create: `pkgs/gctx/test.sh`

- [ ] **Step 1: Write the behavior test**

Create an executable shell test that:

1. Accepts the Babashka executable and `gctx.clj` path as arguments.
2. Creates fake `gcloud` and `fzf` commands in a temporary `PATH`.
3. Makes fake `gcloud config configurations list --format=json` return one
   inactive and one active configuration.
4. Asserts `--help` contains the command description.
5. Asserts `--current` prints the active configuration.
6. Makes fake `fzf` select the inactive configuration and asserts fake
   `gcloud` receives `config configurations activate development`.

Use:

```sh
#!/bin/sh
set -eu

bb=$1
script=$2
tmp=$(mktemp -d)
trap 'rm -rf "$tmp"' EXIT

cat > "$tmp/gcloud" <<'EOF'
#!/bin/sh
if [ "$*" = "config configurations list --format=json" ]; then
  printf '%s\n' '[{"name":"development","is_active":false,"properties":{"core":{"project":"dev-project","account":"dev@example.com"}}},{"name":"production","is_active":true,"properties":{"core":{"project":"prod-project","account":"prod@example.com"}}}]'
elif [ "$*" = "config configurations activate development" ]; then
  printf '%s\n' "$*" > "$GCTX_ACTIVATION_LOG"
else
  exit 1
fi
EOF

cat > "$tmp/fzf" <<'EOF'
#!/bin/sh
grep 'development'
EOF

chmod +x "$tmp/gcloud" "$tmp/fzf"
export PATH="$tmp:$PATH"
export GCTX_ACTIVATION_LOG="$tmp/activation"

"$bb" "$script" --help | grep -F "gctx - gcloud named configurations switcher"
test "$("$bb" "$script" --current)" = "production"
test "$("$bb" "$script")" = "switched to development"
test "$(cat "$GCTX_ACTIVATION_LOG")" = "config configurations activate development"
```

- [ ] **Step 1: Add the package derivation**

Create `pkgs/gctx/default.nix`:

```nix
{
  lib,
  stdenvNoCC,
  makeWrapper,
  babashka,
  fzf,
  google-cloud-sdk,
}:
stdenvNoCC.mkDerivation {
  pname = "gctx";
  version = "0.1.0";

  src = ./.;

  nativeBuildInputs = [ makeWrapper ];

  dontBuild = true;

  installPhase = ''
    runHook preInstall

    install -Dm755 gctx.clj "$out/bin/gctx"
    wrapProgram "$out/bin/gctx" \
      --prefix PATH : "${lib.makeBinPath [
        babashka
        fzf
        google-cloud-sdk
      ]}"

    runHook postInstall
  '';

  meta = with lib; {
    description = "Google Cloud named configuration switcher";
    mainProgram = "gctx";
    platforms = platforms.unix;
  };
}
```
